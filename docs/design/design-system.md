# Atlas Design System

> Version: 2.0
> Status: Active

---

# Purpose

This document defines the visual identity and semantic design principles of Atlas.

Atlas uses **Stitch MCP** as its primary UI generation engine.

This document intentionally does **not** define layouts or page compositions.

Instead, it defines the visual language, branding, and business semantics that every generated interface must follow.

---

# Design Philosophy

Atlas is a **Personal Financial Operating System (PFOS)**.

The interface should communicate:

- Trust
- Simplicity
- Financial clarity
- Calmness
- Confidence

Users should understand their finances—not learn the interface.

---

## Atlas Design System

- Financial Visualization
- Writing Style

---

# Brand Personality

Atlas should feel:

- Professional
- Calm
- Minimal
- Intelligent
- Modern
- Reliable

Avoid interfaces that feel:

- Playful
- Noisy
- Corporate ERP
- Banking Portal
- Gaming Dashboard

---

# Visual Tone

Preferred characteristics:

- Clean spacing
- Minimal borders
- Soft elevation
- Rounded corners
- High readability
- Comfortable whitespace

The interface should never feel crowded.

---

# Color Philosophy

Color communicates meaning.

Never use color as decoration.

---

## Semantic Colors

Green

Meaning

- Income
- Profit
- Positive Growth
- Success

---

Red

Meaning

- Expense
- Debt
- Loss
- Error

---

Blue

Meaning

- Information
- Neutral Actions

---

Orange

Meaning

- Warning
- Pending
- Upcoming Bills

---

Gray

Meaning

- Neutral
- Disabled
- Secondary Information

---

# Category Colors

Transaction categories should **not** depend on colors.

Instead use:

- Icons
- Labels
- Grouping

This keeps dashboards visually calm.

---

# Typography Philosophy

Typography is the primary communication layer.

Hierarchy should always be established through:

- Size
- Weight
- Spacing

Not excessive color.

Numbers should remain highly readable.

---

# Financial Numbers

Money is Atlas's most important content.

Rules:

- Always align comparable numbers consistently.
- Display currency uniformly.
- Distinguish positive and negative values clearly.
- Avoid unnecessary decimal precision.

Financial values should be immediately scannable.

---

# Currency Formatting

Preferred examples:

Rp 125.000

Rp 2.350.000

Never mix multiple formatting styles.

---

# Icon Philosophy

Icons reinforce meaning.

Icons never replace text.

Examples:

Wallet

Savings

Transfer

Debt

Investment

Calendar

Friends

Icons should remain consistent throughout the application.

---

# Financial Visualization

Charts exist to answer questions.

Examples:

Where did my money go?

How much did I save?

Am I spending more than last month?

Avoid decorative charts.

Every visualization should support decision-making.

---

# Motion Philosophy

Motion should communicate:

- Success
- Progress
- Navigation
- State Change

Motion should never exist purely for decoration.

Animations should remain subtle.

---

# Writing Tone

Atlas communicates using:

- Clear language
- Friendly tone
- Professional wording

Avoid technical jargon whenever possible.

---

Example

Good

"Transfer completed successfully."

Bad

"Transaction mutation executed."

---

# Empty States

Empty states should:

- Explain why
- Encourage action
- Reduce uncertainty

Avoid empty pages.

---

# Error Messages

Errors should explain:

- What happened
- What users can do next

Never expose internal system details.

---

# Accessibility

Atlas aims to be accessible by default.

Requirements:

- High contrast
- Keyboard navigation
- Screen reader compatibility
- Touch-friendly controls

Accessibility is a design requirement—not an enhancement.

---

# Dark Mode

Dark mode is a first-class experience.

Do not simply invert colors.

Maintain:

- Contrast
- Hierarchy
- Readability

---

# AI + Stitch Rules

When generating interfaces:

1. Use Stitch MCP as the primary UI generator.
2. Do not redesign layouts manually unless necessary.
3. Preserve Atlas brand identity.
4. Apply Atlas semantic colors.
5. Respect financial visualization rules.
6. Keep generated interfaces consistent with existing screens.

---

# Design Checklist

Before approving any screen:

- Brand identity preserved
- Semantic colors used correctly
- Financial data emphasized
- Numbers easy to compare
- Accessibility maintained
- Writing tone consistent
- Dark mode supported

---

# Guiding Principle

> Atlas does not design screens.

> Atlas designs trust.

Every visual decision should increase user confidence in their financial information.
